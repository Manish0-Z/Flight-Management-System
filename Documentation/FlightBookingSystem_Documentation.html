Flight Booking System Documentation

Introduction

The Flight Booking System is a comprehensive Java-based application designed to manage flight bookings, customer information, and flight data. It features a graphical user interface (GUI) for both administrators and customers, allowing for efficient management of flights, bookings, and user accounts.

Project Structure

The project is organized into the following main packages:

  • bcu.cmp5332.bookingsystem.main: Contains the main entry point and core exception handling
  • bcu.cmp5332.bookingsystem.model: Defines the core data models (Flight, Customer, Booking, User, FlightBookingSystem)
  • bcu.cmp5332.bookingsystem.data: Handles data persistence and management for flights, customers, bookings, and users
  • bcu.cmp5332.bookingsystem.commands: Implements the command pattern for various operations (add, cancel, list, etc.)
  • bcu.cmp5332.bookingsystem.gui: Provides the graphical user interface components

Main Classes

Model Classes

  • FlightBookingSystem: Central class managing all system data and operations
  • Flight: Represents flight information including flight number, origin, destination, departure date, capacity, and price
  • Customer: Represents customer details and their bookings
  • Booking: Represents a booking with associated flight, customer, seat number, and booking date
  • User: Represents system users with authentication details

Data Management Classes

  • FlightDataManager: Handles loading and storing flight data
  • CustomerDataManager: Manages customer data persistence
  • BookingDataManager: Handles booking data operations
  • UserDataManager: Manages user authentication data
  • FlightBookingSystemData: Coordinates all data managers

Command Classes

The system uses the Command pattern for operations:

  • AddBooking: Creates new flight bookings
  • AddCustomer: Adds new customers to the system
  • AddFlight: Adds new flights
  • CancelBooking: Cancels existing bookings
  • EditBooking: Modifies booking details
  • ListCustomers: Displays customer information
  • ListFlights: Shows available flights
  • ShowCustomer: Displays detailed customer information
  • ShowFlight: Shows flight details
  • UpdateFlight: Modifies flight information

GUI Components

Main Windows

  • MainWindow: Entry point for the application
  • AdminMainWindow: Administrator interface for system management
  • CustomerMainWindow_new: Enhanced customer interface with modern design
  • RoleSelectionWindow: Allows users to select their role (admin/customer)
  • LoginDialog: Authentication dialog

Dialog Windows

  • AddBookingWindow: Interface for creating new bookings
  • AddCustomerWindow: Customer registration form
  • AddFlightWindow: Flight creation interface
  • CancelBookingWindow: Booking cancellation interface
  • CustomerBookingDialog: Customer booking creation dialog
  • UpdateFlightWindow: Flight modification interface

Utility Components

  • AirlineLogo: Custom logo component
  • AirplaneIcon: Airplane icon component
  • DesignConstants: UI design constants and utilities
  • ModernHeader: Modern header component
  • RoleIcon: Role-specific icons
  • SidebarIcon: Sidebar navigation icons
  • ToastNotification: Notification system

Key Features

  • User Authentication: Role-based access for administrators and customers
  • Flight Management: Add, update, and view flight information
  • Customer Management: Register and manage customer accounts
  • Booking System: Create, view, and cancel flight bookings
  • Data Persistence: Stores data in text files (flights.txt, customers.txt, bookings.txt, users.txt)
  • Modern GUI: Intuitive graphical interface with sidebar navigation
  • Search Functionality: Search through flights and bookings
  • Toast Notifications: User feedback system

How to Run

  1. Ensure Java is installed on your system
  2. Navigate to the project directory
  3. Compile the project: javac -cp "src" src/bcu/cmp5332/bookingsystem/main/Main.java
  4. Run the application: java -cp "src" bcu.cmp5332.bookingsystem.main.Main
  5. Alternatively, use the provided run.bat file

Data Files

The system uses the following data files located in the resources/data directory:

  • flights.txt: Stores flight information
  • customers.txt: Contains customer data
  • bookings.txt: Holds booking records
  • users.txt: User authentication data

Architecture Notes

  • The system follows the Model-View-Controller (MVC) pattern
  • Data access is abstracted through DataManager classes
  • GUI components are designed with reusability in mind
  • The Command pattern allows for extensible operations
  • Exception handling is centralized in FlightBookingSystemException
